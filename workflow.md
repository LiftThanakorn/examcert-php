# ExamCert Operational Workflow

ระบบ ExamCert มีขั้นตอนการทำงานหลักแบ่งออกเป็น 4 ระยะ ดังนี้:

---

## 1. การเตรียมความพร้อม (Setup Phase)
โดยเจ้าหน้าที่ (Admin) ดำเนินการผ่านระบบหลังบ้าน:

1.  **สร้างโครงการสอบ (Create Project)**:
    - กำหนดชื่อโครงการ, รหัสโครงการ (เช่น RERU-2569-001)
    - ตั้งค่าเกณฑ์คะแนนผ่าน (%), เวลาสอบ (นาที), จำนวนครั้งที่สอบได้
    - เลือก Template เกียรติบัตรที่ต้องการ
2.  **จัดการข้อสอบ (Question Bank)**:
    - เพิ่มข้อสอบรายข้อ หรือนำเข้าแบบ Bulk
    - **สำคัญ**: ต้องตรวจสอบสถานะ "เปิดใช้งาน" (is_active) ของข้อสอบทุุกข้อ
3.  **จัดการผู้มีสิทธิ์สอบ (Participant Whitelist)**:
    - นำเข้ารายชื่อผ่านไฟล์ Excel หรือเพิ่มรายชื่อรายคน
    - ระบบจะสร้าง Access Token 6 หลักให้ผู้สอบแต่ละท่านโดยอัตโนมัติ

---

## 2. การดำเนินการสอบ (Examination Phase)
ผู้สอบ (Participant) ดำเนินการผ่านหน้าเว็บสาธารณะ:

1.  **การเข้าสู่ระบบ (Entry)**:
    - ผู้สอบเลือกโครงการ หรือเข้าผ่านลิงก์โครงการโดยตรง
    - ค้นหาชื่อตนเอง และกรอกรหัส Access Token 6 หลัก
2.  **การทำข้อสอบ (Take Exam)**:
    - ระบบเริ่มนับถอยหลังเวลาสอบ
    - มีระบบ **Auto-save** บันทึกคำตอบลงฐานข้อมูลทันทีที่มีการเลือกข้อ (AJAX) เพื่อป้องกันข้อมูลหายหากอินเทอร์เน็ตหลุด
    - ผู้สอบสามารถข้ามข้อสอบไปมาได้ผ่านทาง Navigator ด้านข้าง
3.  **การส่งข้อสอบ (Submission)**:
    - เมื่อกดส่ง หรือหมดเวลา ระบบจะคำนวณคะแนนทันที
    - ระบบบันทึก Answer Logs ทุกข้อเพื่อใช้ตรวจสอบย้อนหลัง

---

## 3. การสรุปผลและออกเกียรติบัตร (Result & Certification Phase)

1.  **การแสดงผล (Result View)**:
    - ผู้สอบเห็นคะแนนและสถานะ ผ่าน/ไม่ผ่าน ทันทีหลังส่ง
2.  **การออกเกียรติบัตร (Auto-Issuance)**:
    - หากสอบผ่าน ระบบจะเปิดปุ่มให้ "ดาวน์โหลดเกียรติบัตร"
    - ระบบจะ Generate ไฟล์ PDF พร้อม QR Code สำหรับตรวจสอบความถูกต้อง
3.  **การตรวจสอบ (Verification)**:
    - บุคคลภายนอกสามารถสแกน QR Code หรือกรอก Token เพื่อตรวจสอบความถูกต้องของเกียรติบัตรผ่านหน้า Verify สาธารณะได้

---

## 4. การจัดการข้อมูลและรายงาน (Admin Reporting)

1.  **การส่งออกข้อมูล (Export)**:
    - Admin สามารถ Export รายชื่อผู้สอบพร้อมคะแนน และสถานะผ่าน/ไม่ผ่าน ออกมาเป็นไฟล์ Excel ได้
2.  **การบริหารจัดการ (Management)**:
    - Admin สามารถตรวจสอบ Session การสอบที่กำลังดำเนินการอยู่ (Heartbeat) และสามารถสั่งสิ้นสุด Session ได้หากจำเป็น
